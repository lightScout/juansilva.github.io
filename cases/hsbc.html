// THE BRIEF

First Direct wanted a cleaner answer to "where did my money go?" on Android — and an authentication system that wasn't tied to a single device. Two quiet but non-trivial rewrites.

I joined the mobile team to own the Transaction Enrichment and Pulse Dashboard libraries, then stayed on to redesign the in-app chat, integrate international payments, and replace auth end-to-end.

Kotlin throughout. MVVM + Clean Architecture, Hilt for DI, Room where we needed local state, coroutines + RxJava in the legacy seams. JUnit and Mockito all the way down.

Agile Scrum — shipped behind feature flags, measured in production, iterated in two-week loops. The pattern's still the team's default.

International Payments

Integrated cross-border transfers directly into the app — one flow, native controls, no web view bail-out.

PIN-based Auth

Replaced a device-bound auth system with portable PIN-based auth. Fewer "I lost my phone" support calls, more flexibility for customers.

05

24/7 In-App Chat

Redesigned the help surface to make live chat the default — reducing call-centre load and giving customers answers at midnight.

Unit + integration + UI coverage on every shipped library. JUnit, Mockito, Espresso. No "tested in manual" in the PR description.
